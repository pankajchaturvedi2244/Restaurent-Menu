## ‚úî Edge Cases Handled

### ‚úî Email / OTP Handling Note

The application uses **Mailtrap** for email delivery during development/testing.
Because Mailtrap requires an authenticated workspace, **other users will not receive emails or OTPs using my credentials**. They can test the email/OTP flow by:

* Creating their own free Mailtrap account
* Generating SMTP credentials
* Replacing the environment variables in `.env` accordingly

This ensures the email system works safely without exposing my credentials.

I accounted for several real-world scenarios:

* Empty categories or no dishes
* Invalid IDs in API requests
* Missing required fields in POST/PATCH
* Unexpected DB errors
* Network or API failures (fallback UI)
* Loading states for slower connections
